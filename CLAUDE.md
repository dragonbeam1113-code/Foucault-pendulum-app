# CLAUDE.md

## 1. 参照すべき要件

このプロジェクトでは、README.mdに書かれた要件定義に従って実装してください。

特に、目的、MVP、入力、処理、出力、やらないこと、完了条件を重視してください。

## 2. 技術スタック

このプロジェクトでは、次の技術を使ってください。

- Next.js
- App Router
- React
- TypeScript
- Tailwind CSS

外部ライブラリは、明示的に指示されない限り追加しないでください。

## 3. 実装範囲

最初のMVPでは、主に app/page.tsx に実装してください。

実装する範囲は次の通りです。

- 緯度入力欄
- 回転周期の計算
- 結果表示
- 北半球・南半球・赤道付近に応じた説明文
- 簡単なレイアウト

複数ページ構成にはしないでください。

## 4. 実装方針

まずは、1ファイルでシンプルに動く実装を優先してください。

緯度の入力値はReactのstateで管理してください。

回転周期は次の式で計算してください。

- periodHours = 24 / Math.abs(Math.sin(latitude * Math.PI / 180))

緯度0度付近では、計算結果として巨大な数値を表示するのではなく、「赤道付近では見かけの回転はほぼ起こりません」と表示してください。

## 5. 禁止事項

次のことは、明示的に指示されない限り行わないでください。

- データベースを追加しない
- ログイン機能を追加しない
- 外部APIを使わない
- 不要なライブラリを追加しない
- 複雑なディレクトリ構成にしない
- 3D表示を追加しない
- アニメーションを追加しない
- 仕様にない機能を勝手に追加しない

## 6. コーディング方針

コードは、初学者が読んでも理解しやすいようにしてください。

- 変数名は意味がわかるものにする
- 関数名は処理内容がわかるものにする
- 過剰な抽象化を避ける
- コンポーネントを大きくしすぎない
- まずは動くMVPを優先する
- コメントは必要最小限にする

## 7. 安全ルール

このプロジェクトでは、秘密情報を扱わない前提です。

次の情報をコードに直書きしないでください。

- APIキー
- パスワード
- トークン
- データベース接続文字列
- 個人情報

また、秘密情報に NEXT_PUBLIC_ を付けないでください。

明示的に指示されない限り、.env.local を作成・編集しないでください。

## 8. 確認方法

実装後は、次の動作を確認してください。

- 0度を入力すると、赤道付近ではほぼ回転しないと表示される
- 40.6度を入力すると、約37時間前後と表示される
- 90度を入力すると、約24時間前後と表示される
- -90度を入力すると、南極付近の説明が表示される
- 入力値が -90〜90 の範囲で扱われる
- npm run build が通る

## 9. 追加機能：代表都市表示

README.mdに書かれた「入力した緯度に近い代表都市の表示」機能を実装してください。

### 実装方針

* 外部APIは使わない
* アプリ内に代表都市データを固定配列として用意する
* 入力された緯度と都市の緯度の差を計算する
* 緯度差が小さい順に並べる
* 上位3件程度を表示する
* 地理的な最寄り都市ではなく、緯度が近い代表都市として表示する

### 都市データ

最初のMVPでは、次のような固定データを使ってください。

* 弘前市，日本：40.6
* 青森市，日本：40.8
* ニューヨーク，アメリカ：40.7
* ローマ，イタリア：41.9
* 東京，日本：35.7
* 大阪，日本：34.7
* カイロ，エジプト：30.0
* ホノルル，アメリカ：21.3
* シンガポール：1.3
* キト，エクアドル：-0.2
* シドニー，オーストラリア：-33.9
* ブエノスアイレス，アルゼンチン：-34.6
* ケープタウン，南アフリカ：-33.9
* ウシュアイア，アルゼンチン：-54.8

### 禁止事項

* 外部APIを追加しない
* データベースを追加しない
* 現在地取得を追加しない
* 経度を使った本格的な距離計算を追加しない
* 複雑な検索機能にしない

### コーディング方針

* まずは app/page.tsx 内、または必要最小限のファイルで実装する
* 都市データは読みやすい配列として定義する
* 緯度差を計算する関数名はわかりやすくする
* 表示は初学者が読めるシンプルな構造にする
* 既存の周期計算機能を壊さない

### 確認方法

* 40.6度を入力すると、弘前市または青森市が上位に表示される
* 35.7度付近を入力すると、東京が上位に表示される
* 0度付近を入力すると、キトやシンガポールが上位に表示される
* -34度付近を入力すると、シドニー、ブエノスアイレス、ケープタウンが上位に表示される
* npm run build が通る
