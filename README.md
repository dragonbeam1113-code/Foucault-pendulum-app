# Foucault Pendulum Mini App

フーコーの振り子の原理を直感的に理解するための小さなNext.jsアプリです。

## 1. 目的

このWebページの目的は、フーコーの振り子の見かけの回転周期が緯度によって変わることを、数値と短い説明によって直感的に理解できるようにすることです。

## 2. ユーザー

主なユーザーは、フーコーの振り子を初めて学ぶ人です。

具体的には、大学生、物理に関心のある人、フーコーの振り子の展示を見る人を想定します。

## 3. 課題

フーコーの振り子は、地球の自転によって振動面が回転して見える現象ですが、緯度によって回転の速さが変わることを直感的に理解しにくいという課題があります。

特に、赤道ではほとんど回転せず、極では約24時間で一周するという違いを、数式だけでなく入力と結果を通じて確認できるようにします。

## 4. MVP

最小版では、緯度を入力すると、フーコーの振り子の見かけの回転周期を表示できればよいとします。

最初のMVPでは、アニメーションや3D表示は作らず、入力・計算・結果表示に絞ります。

## 5. 入力

ユーザーが入力する値は緯度です。

- 入力項目：緯度
- 単位：度
- 範囲：-90〜90
- 例：0、40.6、90、-90

## 6. 処理

入力された緯度を使って、フーコーの振り子の見かけの回転周期を計算します。

計算式は、学習用の近似として次を使います。

- periodHours = 24 / Math.abs(Math.sin(latitude * Math.PI / 180))

ただし、緯度が0度付近の場合は、sin(緯度)がほぼ0になり、周期が非常に大きくなるため、「赤道付近では見かけの回転はほぼ起こりません」と表示します。

## 7. 出力

画面には次の内容を表示します。

- 入力した緯度
- 見かけの回転周期
- 北半球・南半球・赤道付近に応じた説明文

表示例：

- 緯度0度：赤道付近では見かけの回転はほぼ起こりません
- 緯度40.6度：約37時間で一周して見えます
- 緯度90度：約24時間で一周して見えます

## 8. やらないこと

今回のMVPでは、次の機能は作りません。

- 3Dシミュレーション
- アニメーション
- データベース
- ログイン機能
- 外部API連携
- ユーザーごとの保存機能
- 複数ページ構成
- 高度なグラフ表示

## 9. 完了条件

次の条件を満たせば、MVPは完成とみなします。

- 緯度を入力できる
- -90〜90の範囲の値を扱える
- 0度を入力すると、赤道付近ではほぼ回転しないと表示される
- 40.6度を入力すると、約37時間前後と表示される
- 90度を入力すると、約24時間前後と表示される
- 北半球・南半球・赤道付近で説明文が変わる
- npm run build が通る

## 追加機能：入力した緯度に近い代表都市の表示

### 目的

入力した緯度が、地球上のどのあたりの緯度に相当するのかを直感的に理解できるようにする。

フーコーの振り子の見かけの回転周期は緯度によって変わるため、数値だけでなく、代表的な都市と対応づけることで理解しやすくする。

### MVP

緯度を入力すると、その緯度に近い代表都市を表示する。

### 入力

* 緯度
* 単位：度
* 範囲：-90〜90

### 処理

アプリ内に代表都市と緯度のデータを用意する。

入力された緯度と各都市の緯度の差を計算し、差が小さい順に代表都市を表示する。

計算方法：

* difference = Math.abs(inputLatitude - city.latitude)

最初のMVPでは、外部APIは使わず、アプリ内に固定の都市データを持たせる。

### 出力

入力した緯度に近い代表都市を、緯度差が小さい順に表示する。

表示する情報：

* 都市名
* 国・地域名
* 都市の緯度
* 入力緯度との差

表示例：

* 弘前市，日本：北緯40.6度
* ニューヨーク，アメリカ：北緯40.7度
* ローマ，イタリア：北緯41.9度

### 注意

この機能は、地理的に最も近い都市を表示するものではない。

入力された緯度に近い緯度をもつ代表都市を表示する機能である。

### やらないこと

* 外部APIで都市データを取得しない
* 現在地取得はしない
* 経度を使った距離計算はしない
* 厳密な地理的最寄り都市検索はしない

### 完了条件

* 緯度を入力すると、近い緯度の代表都市が表示される
* 40.6度付近を入力すると、弘前市が上位に表示される
* 0度付近を入力すると、赤道付近の都市が表示される
* 90度や-90度付近では、該当する都市が少ないことを自然に表示できる
* 外部APIを使っていない
* npm run build が通る
